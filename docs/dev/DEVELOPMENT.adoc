Ensure JDK versions are properly configured in `gradle.properties`:

[source,properties]

  ~/.jdks/azul-1.8.0_462,\
  ~/.jdks/azul-17.0.17,\
  ~/.jdks/openjdk-25

----

== Project Architecture Overview

=== Module Structure

_fs_ follows a modular architecture with clear separation of concerns:

* **fs-annotation**: Custom annotations for code analysis and validation
* **fs-asm**: Embedded ASM framework for bytecode manipulation
* **fs-core**: Main library with comprehensive utilities and interfaces
* **fs-all**: Aggregated jar containing all modules (zero-dependency)

=== Module Dependency Graph

[graphviz]
----
digraph G {
    rankdir=TB;

----

=== Multi-Version JDK Support

The project uses `FsLoader` to load appropriate implementations based on runtime JDK version:

[source,java]
----
interface HttpCallerService {
    @Nonnull
    HttpCallerService INST = FsLoader.loadImplByJvm(HttpCallerService.class, 11);

    @Nonnull
    HttpCaller newCaller(int bufSize, @Nonnull Proxy proxy) throws HttpNetException;
}
----

=== Design Principles

_fs_ is built on the following design principles:

* **Zero Dependency**: Minimal external dependencies, self-contained implementations
* **Multi-Version Support**: Automatic JDK version detection and optimal implementation selection
* **High Performance**: Optimized implementations with benchmark validation
* **Modularity**: Clear module boundaries and responsibilities
* **Extensibility**: Well-defined interfaces for custom implementations
* **Consistency**: Unified API design and naming conventions
* **Reliability**: Comprehensive test coverage and error handling
* **Simplicity**: Intuitive APIs and clear documentation

== Module Details

=== fs-annotation

src: link:../../fs-annotation/[fs-annotation]

Provides custom annotations for code analysis, validation, and static analysis support.

==== Key Annotations

* **Null Safety**: `@Nonnull`, `@Nullable` for null pointer prevention
* **Immutability**: `@Immutable` for immutable types
* **Parameter Handling**: `@RetainedParam` for parameter retention
* **Visibility**: `@NonExported` for non-exported packages

==== Usage Guidelines

* Use `@Nonnull` and `@Nullable` consistently for method parameters and return types
* Apply `@Immutable` to value objects and data classes
* Use package-level annotations for default nullability settings

=== fs-asm

src: link:../../fs-asm/[fs-asm]

Embedded ASM framework for bytecode manipulation, providing dynamic programming capabilities without external dependencies.

==== Key Features

* **Zero Dependency**: Complete ASM implementation with no external dependencies
* **Package Isolation**: All classes in `space.sunqian.fs.asm` package
* **Version Compatibility**: ASM 7.1 feature set

==== Core Components

* **Class Manipulation**: `ClassReader`, `ClassWriter`
* **Method Handling**: `MethodVisitor`, `FieldVisitor`
* **Bytecode Analysis**: `AnnotationVisitor`, `Type`
* **Framework Support**: Complete ASM opcodes and utilities

==== Usage in _fs_

* Used by _fs-core_ for dynamic proxy and AOP functionality
* Enables bytecode generation for performance optimization
* Provides reflection alternatives for high-performance scenarios

=== fs-core

src: link:../../fs-core/[fs-core]

Main library with comprehensive utilities and interfaces.

==== Architecture Overview

_fs-core_ is organized into functional packages:

* **Root Package** (`space.sunqian.fs`): Core utilities and global access point
* **Base Utilities** (`space.sunqian.fs.base`): Fundamental utilities
* **Cache** (`space.sunqian.fs.cache`): Lightweight caching implementation
* **Collections & Data** (`space.sunqian.fs.collect`, `space.sunqian.fs.data`)
* **Concurrency** (`space.sunqian.fs.concurrent`): Concurrent programming utilities
* **I/O Operations** (`space.sunqian.fs.io`): Byte/char processing and file operations
* **Networking** (`space.sunqian.fs.net`): HTTP, TCP, UDP with adaptive implementations
* **Object Manipulation** (`space.sunqian.fs.object`): Meta, building, conversion, and pooling
* **Dynamic Programming** (`space.sunqian.fs.dynamic`, `space.sunqian.fs.invoke`)
* **Dependency Injection** (`space.sunqian.fs.di`): Lightweight DI container
* **Reflection** (`space.sunqian.fs.reflect`): Reflection utilities and type handling
* **Third-party Integration** (`space.sunqian.fs.third`): Integration with third-party libraries
* **Utilities** (`space.sunqian.fs.utils`): Codec, event bus, JDBC, versioning

==== Object Manipulation Package

The `space.sunqian.fs.object` package contains key components:

* **meta**: Object metadata definition and introspection
  * `ObjectMeta`: Object metadata for class introspection
  * `MapMeta`: Map metadata for type-safe map operations
  * `PropertyMeta`: Property metadata for individual fields
  * `MetaKit`: Utility methods for metadata operations
* **builder**: Object building with builder patterns
  * `BuilderManager`: Central manager for builder operations
  * `BuilderOperator`: Builder operator interface
* **convert**: Object conversion and property copying
  * `ObjectConverter`: Object conversion interface
  * `ObjectCopier`: Property copying interface
  * `ConvertKit`: Conversion utilities
* **annotation**: Annotations for object-related operations
* **pool**: Object pooling interfaces

==== Key Design Patterns

* **Factory Pattern**: Used extensively for creating instances with version-specific implementations
* **Builder Pattern**: Network clients and object builders use fluent builder interfaces
* **Strategy Pattern**: Different implementations selected based on runtime environment
* **Adapter Pattern**: JDK version compatibility through adapter implementations

==== Development Guidelines

* Place kits, interfaces, and implementations in appropriate packages
* Use `FsLoader` for multi-version JDK support
* Follow the established naming conventions and package structure
* Write comprehensive tests for all new functionality
* Maintain interface-back-implementation pattern for extensibility

== FAQ

=== How to add a new utility class?

1. Place in appropriate package under `fs-core/src/main/java`
2. Follow naming conventions (e.g., `XxxKit`)
3. Add unit tests
4. Update documentation

=== How to support a new JDK version?

1. Add JDK path to `gradle.properties`
2. Create version-specific implementation using `FsLoader`
3. Add tests for the new version
4. Update build configuration

=== How to add new object conversion handlers?

1. Implement handler in `space.sunqian.fs.object.convert.handlers`
2. Register handler in appropriate converter/copier
3. Add tests for the new handler
4. Update documentation if needed

=== Why is fs-asm excluded from test coverage?

`fs-asm` is a direct copy of the ASM framework with package relocation. No modifications to original ASM functionality, so testing is handled by the ASM project itself.
